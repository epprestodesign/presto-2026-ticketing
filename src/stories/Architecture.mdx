# Architecture & Conventions

The machine-oriented map of this library. If you're an **AI agent** extending or
editing the system, read this first — it tells you **where things live, what to
reuse, and the conventions to match** so new work is indistinguishable from
existing work.

---

## Reading the system efficiently

1. **Find the surface** in the sidebar — primitives (`Foundations … Navigation`)
   or experience flows (`Browse Hotels … Manage Booking`). See **Introduction**
   for the category map.
2. **Locate the source** — every story maps 1:1 to a component (table below).
3. **Reuse before rebuilding** — check the *Shared building blocks* index; most
   new screens compose existing pieces.
4. **Match the conventions** — tokens, naming, and story shape below.

## Design tokens & theming

- **Always use DS tokens**, never hardcoded hex. Tokens are CSS custom properties
  named `--ds-*`:
  - Palette ramps: `--ds-palette-<hue>-<step>` (e.g. `--ds-palette-navy-900`, `--ds-palette-emerald-600`).
  - Semantic roles: `--ds-color-text`, `--ds-color-text-subtle`, `--ds-color-surface`, `--ds-color-border`, `--ds-color-border-bold`, `--ds-color-background-brand-bold`, `--ds-color-text-danger`, `--ds-color-text-success`.
  - Scales: `--ds-radius-*`, `--ds-duration-fast`, `--ds-ease-standard`, `--ds-shadow-*`.
- **Primary brand = Navy** (#01113E). Reach for `--ds-color-background-brand-bold`
  for actions and `--ds-color-text-brand` for brand headings.
- Page backgrounds use `--ds-palette-slate-100`; foreground cards use
  `--ds-color-surface` with a `--ds-color-border` outline (the checkout/account pattern).

## Component conventions

- **Vue 3 SFC, `<script setup>`.** Props via `defineProps`, events via
  `defineEmits`, imperative API via `defineExpose`.
- **Quasar is globally registered** — use `<q-*>` tags directly (no per-file
  import). Plugins available: `Notify`, `Dialog`, `Loading` (see `preview.js`).
- **Scoped styles**, BEM-ish naming: one **prefix per component**, then
  `prefix__element` / `prefix--modifier` (e.g. `.hlc__media`, `.mb__rowbox`,
  `.flt__title`). Teleported content (menus, notifies) needs an **unscoped**
  `<style>` block.
- **State sync** patterns in use: `v-model:<name>` (e.g. `v-model:open`),
  parent-owned reactive lists passed as props + `remove`/`update` events, and
  `defineExpose({ … })` for imperative APIs like `clear()`.
- **Modals & panels are primitives.** Centered dialogs use **`DsModal`**
  (backdrop, ESC, click-out, scroll-lock, mobile-fullscreen, `#header`/`#footer`
  slots, teleported to `<body>`); slide-overs use **`DsSidePanel`**
  (`side="right" / "left" / "center"`). Don't hand-roll overlay chrome — the
  booking dialogs (`AvailabilityDialog`, `AllFiltersDialog`) and the
  flyouts/sheets (`CartFlyout`, `ProfileEditModal`) are built on these.
- **Amenities come from one catalog.** Reference `src/lib/amenities.js` by `key`
  and render via the `Amenity` component (or the helpers `amenityGroups()` /
  `filterAmenities()` / `popularAmenities()`). Don't hardcode amenity lists in
  components or stories.
- Match the **comment density and idiom** of the file you're editing.

## Story conventions

- One `*.stories.js` per component. `title: 'Category/Name'` decides placement;
  add `tags: ['autodocs']` for a generated Docs tab.
- **Variants are named exports** (`export const GroupHold = …`). Full-page
  components set `parameters: { layout: 'fullscreen' }`.
- Live/interactive demos use a `render()` with a `setup()` returning reactive
  state (see `Browse Hotels / Booking Widget`).
- Sidebar order is **explicit** in `preview.js` `storySort` — add new categories
  there (anything after `'*'` sorts last, e.g. `Manage Booking`).

## Adding a component (checklist)

1. Create the SFC in `src/components/` (or the right `<area>/`).
2. Use DS tokens + the prefix naming convention; reuse a shared block if one fits.
3. Add `src/stories/<category>/Name.stories.js` with `title`, `tags: ['autodocs']`, and variants.
4. If it's a new category, add it to `storySort` in `.storybook/preview.js`.
5. Verify with `pnpm build-storybook` (the deploy gate).

## Gotchas

- **`<filter>` is a native SVG tag** — register components under a distinct tag
  (e.g. `<browse-filter>`), or Vue renders an empty SVG element instead of your component.
- **Notify/menus are teleported** — style them from an **unscoped** `<style>` or
  global `app.scss` (e.g. `.ds-toast`, `.srt__menu`).
- **Don't set `position` on a notification** (`.ds-toast`) — Quasar's leave
  transition sets it to `position:absolute`, and an override (same specificity,
  loaded later) breaks the exit animation (it jumps sideways). Its progress bar
  also defaults to `left/right:-10px`; inset it to `0` to stay in the box.
- **Maps need an API key** — `HotelMap` degrades to a placeholder; never assume a key exists.
- **`references/` is local-only** (gitignored) — design screenshots live on disk, not in the repo.
- **Run from a git branch**, not `main`; deploy happens automatically on merge to `main`.
