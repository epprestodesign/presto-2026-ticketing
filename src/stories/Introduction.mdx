# Presto Design System

A **themed [Quasar](https://quasar.dev) component library** for a consumer
**hotel booking & reservation** experience, documented and QA'd in Storybook.

Every example, label, and edge case is written around the booking journey —
searching destinations, comparing rooms and amenities, holding rooms for a group
or making a single reservation, paying, confirming, and managing the booking
afterward. The library spans the **mobile booking app, desktop web, and
host/admin surfaces**.

> **Building with an AI agent?** Read **Getting Started → Architecture &
> Conventions** first — it's the machine-oriented map (task → location,
> conventions, and the shared components to reuse instead of rebuild).

---

## How to read this library

The sidebar is organized the way product and design think, not alphabetically.
It has **two layers**:

**1. Experience flows** — the booking journey, in order:

`Browse Hotels` → `Hotel Details` → `Checkout Experience` → `Confirmation` → `Manage Booking`

**2. Primitives** — the design-system building blocks every flow is assembled
from: **Foundations, Inputs, Data Display, Feedback, Layout, Navigation**.

Three recurring **booking modes** run through the flows and you'll see them named
everywhere:

- **`reserve`** — a single **hotel reservation** (one stay, check-in/out, one guest party).
- **`hold`** — **holding rooms for a group or team** (multiple hotels, rooms held per night via quantity steppers, teams/block).
- **`reservations`** — **multiple room reservations** at once (several hotels → rooms → booked nights, shown as date + rate rows; no quantity selector).

## Experience flows

### Browse Hotels
**Discover and choose where to stay.** Search a destination, filter results,
compare listings on cards or a map, and start a booking.

- **Booking Widget** — destination + dates + guests search entry.
- **Hotel Listing Card** — the core result card (**horizontal**; the vertical
  orientation is deprecated), with a photo carousel, rating, price, and a
  **group-booking** mode that adds an **Availability** secondary button opening
  the **Availability Dialog** (hotel summary + a `hold`-mode rooms carousel).
- **Hotel Map** — map-only results with price-pill markers and clustering (with a
  zoom cutoff so pins de-cluster as you zoom in), native **+/- zoom** controls, and
  a **search-radius** circle centered on the event location that's **two-way bound**
  to a slider — zoom or ⌘-scroll the map and the radius follows; drag the slider and
  the map reframes (Google Maps, with a graceful no-key fallback).
- **Search & Filters** — one `Filter` component covering every filter type
  (Amenities list/grid with **inline search**, **Room Type** chips always ordered
  least→max bed occupancy (twin → double → queen → king → suite), Guest Rating,
  Star Rating, Hotel class, **Brands tree with inline search**, Budget range with a
  **per-night ⇄ total** toggle, **editable Min/Max** + price histogram, Property
  Search + autocomplete). Long amenity lists collapse behind a **"View more /
  Fewer"** toggle. An **All Filters** button opens a modal consolidating every
  filter into accordion sections with a live "Show N results" footer.
- **Forms, Empty States, Loading States** — supporting patterns for the flow.

### Hotel Details
**The chosen hotel, end to end.** A composed detail screen and the pieces it's
built from — each is also its own story.

- **Hotel Detail Page** — the full composition: hero gallery → sticky detail tabs
  → summary header → room selection → about / amenities / policies.
- **Photo Gallery** — the hero photo **mosaic** (lead image + cluster) with a
  "See all photos" modal (reuses `DsImageList`).
- **Detail Tabs** — Overview / Rooms / Property / Amenities / Policies section nav.
- **Hotel Summary Header** — name + stars, address, guest score, popular
  amenities, and a mini-map.
- **Rooms** + **Room Card** — the "Select Your Room" **carousel** of room cards
  (`reserve` & `hold` modes; sold-out, limited, discounted, refundable states).
- **Room Booking Dialog** — the per-room booking modal (moved here from Browse).
- **Amenities** + **Policies & Property** — categorized amenities and the
  policies / "about the property" blocks.

### Checkout Experience
**Turn a selection into a booking.** An Airbnb-style stepped flow, with each step
its own component and a sticky order-summary rail.

- **Checkout Page** — orchestrates the steps; **Group Block**, **Reservation**, and
  **Multiple Room Reservations** (`reservations` mode) variants.
- **Steps** — *Review Order*, *Contact Info* (for `hold`, the group/teams block +
  required Group Block Name; for `reserve`, **per-room guest information** that's
  aware of the booking widget's room/traveler selection — team name + event
  custom fields, and additional guests pre-seeded from occupancy; for
  `reservations`, the same per-room forms **grouped by hotel/reservation**),
  *Payment*, *Review Reservation* (protection + cancellation policy).
- **Payment Dialogs** — Instacart-style payment-method and add-card surfaces.

### Confirmation
**The post-booking page.** A `ConfirmationPage` with **Reservation** and **Group
Block** variants: success hero, itinerary, and post-booking modules — experience
**rating**, **who's going**, **cancellation** + change/cancel, **add to
calendar**, **location** (live map), **area tips**, **FAQ**, and **help/contact**.

### Manage Booking
**The account home** (pinned last). A two-column page: avatar + name + Edit
profile on the left with a selectable menu; section content on the right —
**Profile** (Basic info / Contact / More details / Additional travelers with a
full-height edit modal), **Communications** toggles, **Payment methods** (add via
the checkout dialog; edit/delete), **Security**, and **Help & feedback**.

## Primitives (building blocks)

The flows above are assembled from these. Each story has a **Docs** tab with
usage, props, and guidance.

- **Foundations** — the tokens themselves, made visible. Start here to understand
  the palette, type scale, spacing, radius, and motion the whole system uses.
- **Inputs** — every form control, including the booking-specific **Quantity
  Stepper**, plus primitives extracted from the booking flows: **Choice Chips**
  (chip toggle group), **Range** (dual-handle slider), **Checkbox Tree** (parent
  selects children), and **Phone Field** (country-code input).
- **Data Display** — avatars, badges, chips, dividers, icons, lists, tables,
  tooltips, typography, plus **List Item** (the standard row), **Accordion**
  (expand/collapse), **Rating** (read-only review score), and **Amenity** (the
  atomic icon + label unit, sourced from the shared amenities catalog —
  `src/lib/amenities.js`, the single source of truth used by the header,
  amenities section, cards, and filters).
- **Feedback** — alerts, banners (Atlassian-Flag style), dialogs, **Modal**
  (`DsModal` — the centered dialog shell behind the availability / all-filters /
  map dialogs), snackbars, **Toast** (DS-styled Quasar Notify) — all sharing one
  hue convention — progress, skeletons, backdrops.
- **Layout** — `Ds*` structural wrappers: Box, **Card** (the standard surface),
  Container, Grid, Stack, Image List, Section Header, and **Side Panel** (the
  slide-over chrome — right / left / center — behind the cart fly-out and the
  profile edit sheet).
- **Navigation** — Global Nav (brand + Manage Booking + cart, with a fly-out),
  tabs, menus, breadcrumbs, pagination, steppers.

## Where to customize

| What you want to change | File |
| --- | --- |
| Brand colors, base font, global radius | `src/css/quasar.variables.scss` |
| Design tokens (palette, color roles, spacing, radius, type) | `src/css/ds-palette.scss`, `src/css/ds-color-tokens.scss`, `src/css/tokens.scss` |
| Per-component overrides & global CSS (incl. `.ds-toast`) | `src/css/app.scss` |
| Sidebar order / category IA | `.storybook/preview.js` (`storySort`) |
| Amenities (catalog, filter list, popular set) | `src/lib/amenities.js` (single source of truth) |
| Components | `src/components/` (top-level primitives + `browse/`, `details/`, `checkout/`, `confirmation/`, `managebooking/`) |
| Stories | `src/stories/<category>/*.stories.js` |

## Workflow

1. `pnpm storybook` → open <a href="http://localhost:6006" target="_blank">localhost:6006</a>.
2. Edit any file above — the canvas hot-reloads instantly.
3. Use the **Controls** / **Docs** tabs on each story to QA props and variants.
4. When happy, `pnpm build-storybook` produces a static site in `storybook-static/`.

## Deploy

The build output is a plain static site (GitHub Pages, Netlify, any static host).
Pushing to `main` deploys to GitHub Pages via `.github/workflows/deploy.yml`.
See `README.md` for details.
